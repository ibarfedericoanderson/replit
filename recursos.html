Mejores prácticas

Para Claude Sonnet 3.5
  • Sé específico en tus solicitudes para obtener respuestas más precisas
  • Proporciona contexto suficiente sobre tu proyecto y objetivo
  • Divide problemas complejos en consultas más pequeñas y manejables
  • Utiliza ejemplos concretos cuando expliques lo que necesitas
  • Refina iterativamente las respuestas con preguntas de seguimiento
  • Revisa siempre el código generado antes de implementarlo
  • Pide explicaciones cuando no entiendas algo de la respuesta
Para el Asistente de Replit
  • Personaliza las sugerencias de autocompletado según tus preferencias
  • Utiliza atajos de teclado para aceptar o rechazar sugerencias rápidamente
  • Aprovecha las sugerencias proactivas para mejorar tu código
  • Para consultas breves, usa el asistente en lugar de iniciar una conversación con Claude
  • Activa las notificaciones para recibir sugerencias de mejoras
  • Utiliza el asistente para verificar rápidamente la sintaxis
  • Aprovecha su capacidad para identificar errores potenciales en tu código

Prompts efectivos para programación

Estructura recomendada:

  1. Especifica el lenguaje de programación
  2. Describe la funcionalidad deseada con detalle
  3. Menciona requisitos específicos (eficiencia, legibilidad, etc.)
  4. Proporciona ejemplos de entradas y salidas esperadas
  5. Especifica si necesitas comentarios explicativos

Ejemplo:

"Crea una función en Python que ordene un diccionario por valores en orden descendente. La función debe manejar valores numéricos y de texto. Para valores de texto, ordena alfabéticamente. Incluye manejo de errores para entradas no válidas y comentarios explicativos. Ejemplo de uso: ordenar_diccionario({'a': 5, 'b': 2, 'c': 8}) debería devolver {'c': 8, 'a': 5, 'b': 2}"

Estructura recomendada:

  1. Proporciona el código completo o la sección relevante
  2. Describe el error o comportamiento inesperado
  3. Incluye cualquier mensaje de error que estés recibiendo
  4. Explica el comportamiento esperado
  5. Menciona lo que ya has intentado para solucionarlo

Ejemplo:

"Tengo el siguiente código Python que debería leer un archivo CSV, filtrar las filas según ciertos criterios y escribir los resultados en un nuevo archivo. Sin embargo, estoy recibiendo un error 'KeyError: 'column_name'' en la línea X. El código es: [código]. Quiero que filtre las filas donde la columna 'age' tiene un valor mayor que 30. Ya he verificado que el archivo CSV existe y tiene los encabezados correctos."

Estructura recomendada:

  1. Proporciona el código que deseas optimizar
  2. Explica qué aspectos te gustaría mejorar (velocidad, memoria, legibilidad)
  3. Menciona cualquier restricción o requisito que debas mantener
  4. Incluye información sobre el tamaño típico de los datos
  5. Indica el contexto de ejecución (entorno, limitaciones de hardware, etc.)

Ejemplo:

"Necesito optimizar esta función de procesamiento de imágenes en Python para mejorar el rendimiento. Actualmente tarda demasiado con imágenes grandes (4K). Me gustaría reducir el tiempo de procesamiento sin sacrificar significativamente la calidad. El código actual es: [código]. Este código se ejecutará en una Raspberry Pi 4 con 4GB de RAM, así que la eficiencia de memoria también es importante."

Consejo: Recuerda que puedes utilizar Claude Sonnet 3.5 y el Asistente de Replit de forma complementaria. Claude es excelente para tareas complejas y explicaciones detalladas, mientras que el Asistente de Replit brilla en la asistencia en tiempo real durante la codificación.
